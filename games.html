I was originally going to have this be the front page of my site, but decided against it for reasons of sanity. It's unfortunately still a big buggy, performs much better in Chrome than Firefox, and won't work at all on mobile. It was hugely fun to make though, and still interesting (and I spent more time on it than I would like to admit), so I've still included it on the site.

The Pacman game was created using Quintus. The Quintus framework is still in beta, so there are still some issues, but it seems pretty solid. It definitely made creating the basic Pacman game fairly easy. I went through the basic "TowerMan" tutorial to get ideas on how to implement a Pacman style game in the framework, and then extended it to improve the AI and add it to my site.

I manually laid out all of the divs with site content in the grid I created for the map, on a 70x70 pixel grid. (Looking back, it would have been better to do this dynamically with JS, because it's pain to reposition things as is.) The map is stored as a JSON file, which Quintus uses to create the collision layer for the game. Which is how you're able to navigate around the div elements on that page.

Pacman and the ghosts have some basic pathfinding AI using an A* pathfinding algorithm. I followed this tutorial for implementing A* in Python, and then just translated it to Javascript, which was pretty easy. It's efficient except for the fact that I'm manually sorting the elements rather then using a priority queue...it doesn't seem to make a huge performance difference as there are few elements to sort (and a new path is only calculated once every few seconds or so), but ideally I would refactor it to use a priority queue instead.

Currently, Pacman just picks a random dot plot a path towards. To make him more efficient I would want to implement a nearest neighbor search to have him go after near dots first. This is why the Pacman AI can seem pretty erratic at times.

The ghosts also use the same A* algo to chase after Pacman. The only difference is that when Pacman is using it, ghosts are considered "unreachable" elements on the map, so he doesn't plot a path through them. At every game loop, Pacman also checks two tiles in front of him (and tries to look around corners, but that seems kind of broken), and if there's a ghost he calculates a new path in the opposite direction.

This was super fun to make. Given more time, I want to improve the AI and performance.
